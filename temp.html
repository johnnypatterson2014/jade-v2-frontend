Einstein Summation (einsum) Cheat Sheet for Machine Learning

Core Rule

Einstein summation follows two simple rules:

  • Repeated indices are summed over.
  • Indices that remain appear in the output tensor.

Example:

$$ C_{ij} = \sumk A{ik} B_{kj} $$

Einsum pattern:

ik,kj -> ij

Common Einsum Patterns

Operation Formula Einsum Pattern
Dot Product $$x \cdot y = \sum_k x_k y_k$$ k,k ->
Matrix Multiplication $$C_{ij} = \sumk A{ik} B_{kj}$$ ik,kj -> ij
Gram Matrix (Pairwise Dot Products) $$S_{ij} = \sumk X{ik} X_{jk}$$ ik,jk -> ij
Attention Scores $$S_{bts} = \sumd Q{btd} K_{bsd}$$ btd,bsd -> bts
Attention Output $$O_{btd} = \sums A{bts} V_{bsd}$$ bts,bsd -> btd
Multi-Head Attention Scores $$S_{bhts} = \sumk Q{bhtk} K_{bhsk}$$ bhtk,bhsk -> bhts

Mental Model

Most deep learning tensor operations follow the same pattern:

$$ A{\dots k} ; B{\dots k} \rightarrow \sum_k $$

Meaning:

  1. Align tensors along a shared dimension.
  2. Multiply elementwise.
  3. Sum over the shared index.

This produces a matrix or tensor of pairwise interactions.

Key Insight

Most neural network layers reduce to computing many dot products:

  • Linear layers → matrix multiplication
  • Attention → query-key dot products
  • Kernel methods → Gram matrices
  • Convolutions → local dot products

Einstein summation simply makes these interactions explicit.
